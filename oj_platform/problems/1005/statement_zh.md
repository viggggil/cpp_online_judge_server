# 1005. 失落文明（困难版）

时间限制：2 秒  
内存限制：256 MB

## 题目描述

这是题目的困难版本。与简单版本的区别在于：这里需要你计算**所有子段**上的值之和。

定义一种生成长度为 `m + k` 的整数序列的算法：

1. 首先输入一个长度为 `m` 的整数序列 `x`。如果 `k = 0`，算法立刻结束并返回 `x`。
2. 否则任选一个下标 `1 <= i <= |x|`，并在元素 `x_i` 的**后面立即插入**一个新元素，其值为 `x_i + 1`。
3. 当序列长度达到 `m + k` 时结束；否则继续执行第 2 步。

对一个长度为 `n` 的序列 `b`，定义 `f(b)` 为：能够通过上述算法生成 `b` 的**最短初始输入序列长度**。

现在给定一个长度为 `n` 的序列 `a`，你需要计算：

`sum(f([a_l, a_{l+1}, ..., a_r]))`，其中对所有 `1 <= l <= r <= n` 的子段都要计算并求和。

换句话说，要求出序列 `a` 的所有连续子段的 `f` 值之和。

## 输入格式

第一行一个整数 `t`，表示测试组数。  
对于每组测试：

- 第一行一个整数 `n` (`1 <= n <= 300000`)；
- 第二行 `n` 个整数 `a_1, a_2, ..., a_n` (`1 <= a_i <= 10^9`)。

保证所有测试组的 `n` 之和不超过 `300000`。

## 输出格式

对于每组测试，输出一个整数，表示答案。

## 样例输入

```text
5
5
1 2 3 4 5
5
1 3 5 7 9
5
1 2 5 6 5
7
1 2 4 5 3 7 8
9
9 8 9 2 3 4 4 5 3
```

## 样例输出

```text
15
35
25
60
78
```

## 提示

- 如果一个子段内部存在很多形如 `x, x+1` 的局部可消去结构，那么 `f` 值可能显著变小。
- 困难版要求的是**所有子段**的总和，因此需要远优于朴素枚举子段的做法。
